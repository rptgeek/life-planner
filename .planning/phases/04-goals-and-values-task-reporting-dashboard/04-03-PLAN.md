1. Import `useReportData` from `@/lib/hooks`

**Page structure per UI-SPEC:**

- Page title: `<h1 className="text-3xl font-semibold text-slate-800">Reports</h1>` (Display size 28px)
- Subtitle: `<p className="text-sm text-slate-500 mt-1">How your goals are progressing</p>`

**Loading state:** If `loading`, render 3 skeleton cards: `<div className="animate-pulse rounded-lg bg-slate-200 h-24 w-full" />` in a grid.

**Empty states (per UI-SPEC copywriting contract):**
- If goalReports.length === 0 and unlinkedTasks.length === 0 and not loading: show "No active goals yet. Add goals on the Goals page to see your progress here."
- If goalReports.length === 0 but unlinkedTasks.length > 0: show "No tasks found. Schedule tasks from the Daily Planner to see them tracked here." (only for the goal section)

**Main content (when not loading and data exists):**

a) `<ForgottenGoalsCallout goals={forgottenGoals} />` — above cards

b) Section heading "Goal Breakdown" (`text-xl font-semibold text-slate-800 mt-6 mb-4`)
   Grid of GoalReportCards: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6`
   Map goalReports to `<GoalReportCard key={r.goal.id} report={r} />`

c) `<UnlinkedTasksSection tasks={unlinkedTasks} />` with `mt-8`

d) `<TrendChartsSection data={trendData} goalTitles={goalTitles} />` with `mt-8`
   Show empty state text if trendData.length === 0: "Not enough data to show trends yet..." (this is handled inside TrendChartsSection)

**Error handling:** Wrap the hook usage — if an error occurs, show: "Could not load report data. Check your connection and refresh the page." (per UI-SPEC error state copy). Use a try/catch boundary or check for undefined data gracefully.
