## Interaction Contract

### Goal Card

- Each goal renders as a white card with 1px `border-slate-200` border, `rounded-lg`, `p-4` (16px) padding, `shadow-sm`.
- Shows: goal title (Heading, 20px semibold), role badge (Label, 12px semibold, colored chip), category badge (Label, 12px, slate-500).
- Metric row: "Open: N  Completed: N  Overdue: N" — overdue N renders in `text-red-600 font-semibold` only when N > 0.
- Progress bar: full-width `h-2` bar, `bg-slate-200` track, `bg-indigo-500` fill. Width set via inline `style={{ width: '${pct}%' }}`.
- Completion % shown as `text-indigo-600 text-sm font-semibold` below the bar.
- Goals with `status = 'completed'` or `status = 'archived'` are excluded from the main grid by default (no toggle needed — deferred to Claude's discretion; exclude them).
- Cards do not have click targets in this phase — read-only display only.

### Forgotten Goals Callout

- Amber left-border callout: `border-l-4 border-amber-400 bg-amber-50 p-4 rounded-r-lg`.
- Heading: "Goals without tasks (N)" in `text-amber-800 font-semibold text-sm`.
- Lists goal titles inline as comma-separated text in `text-amber-700 text-sm`.
- Only visible when N > 0.

### Unlinked Tasks Section

- Collapsible accordion triggered by a chevron button.
- Header: "Unlinked Tasks (N)" — always visible count.
- When expanded: flat list of task titles, each on its own row, `text-sm text-slate-700`.
- Uses `ChevronDown` / `ChevronRight` from lucide-react for toggle (consistent with goals page pattern).

### Trend Charts Section

- Recharts `ResponsiveContainer` wrapping a `LineChart`.
- One line per goal, each line a distinct indigo shade (use Recharts `stroke` with opacity stepped: 1.0, 0.7, 0.5, 0.3 for up to 4 goals; beyond 4 goals cycle same shades).
- X-axis: weeks (ISO week labels, e.g. "Mar 31", "Apr 7").
- Y-axis: tasks completed count (integer, no decimals).
- Tooltip: shows goal title + count for the hovered week.
- Chart granularity: weekly only (no toggle — Claude's discretion to keep scope contained).
- Chart height: 280px (fixed, not responsive-height).
- Section placed below the GoalBreakdownSection, separated by `mt-8` (32px).

### Loading State

- Each section shows a `animate-pulse` skeleton: a `rounded-lg bg-slate-200 h-24 w-full` placeholder per card slot.
- Number of skeleton cards: 3 (reasonable default while data loads).
