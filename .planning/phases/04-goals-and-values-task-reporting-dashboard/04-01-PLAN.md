<task type="auto">
  <name>Task 2: Create useReportData hook</name>
  <files>src/lib/hooks.ts</files>
  <read_first>src/lib/hooks.ts, src/lib/types.ts</read_first>
  <action>
Add a `useReportData` hook to the END of src/lib/hooks.ts. Import `GoalReport, TrendDataPoint, ReportData` from types.

The hook:
1. Calls `useGoals()` and `useTasks()` (no dateFilter — gets ALL tasks).
2. Filters goals to `status === 'active'` only (per D-14 / UI-SPEC: exclude completed/archived).
3. For each active goal, counts tasks where `task.goal_id === goal.id`:
   - `openCount`: tasks with `completed === false`
   - `completedCount`: tasks with `completed === true`
   - `overdueCount`: tasks with `completed === false && due_date !== null && due_date < today` (compare as ISO date strings, today = `new Date().toISOString().split('T')[0]`)
   - `totalCount`: openCount + completedCount
   - `completionRate`: totalCount > 0 ? Math.round((completedCount / totalCount) * 100) : 0
4. `forgottenGoals`: active goals where no tasks have that goal_id (per D-11).
5. `unlinkedTasks`: tasks where `goal_id === null` (per D-06).
6. `trendData`: For all tasks with `completed === true` and `completed_at !== null`, group by goal title and by ISO week (Monday-start). Build an array of TrendDataPoint objects spanning the last 8 weeks. Each point has `weekLabel` (format: "MMM D" of Monday, e.g. "Mar 31"), `weekStart` (ISO date), and a key per goal title with the count of tasks completed that week. Goals with zero completions across all weeks are omitted from trend data.
7. Return `{ goalReports, forgottenGoals, unlinkedTasks, trendData, loading: goalsLoading || tasksLoading }` typed as ReportData.

Use `useMemo` to memoize all computed arrays (goalReports, forgottenGoals, unlinkedTasks, trendData) with dependencies on `[goals, tasks]`.

Do NOT create a new Supabase query — reuse the existing hooks. The computation is client-side.
