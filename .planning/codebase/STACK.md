## Configuration

**Environment:**
- Loads from `.env.local` (present but contents not examined per security policy)
- Public variables must be prefixed with `NEXT_PUBLIC_`

**Required Environment Variables:**
- `NEXT_PUBLIC_SUPABASE_URL` - Supabase project URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` - Supabase anonymous key for client-side auth
- `NEXT_PUBLIC_GOOGLE_CLIENT_ID` - Google OAuth client ID for Calendar integration

**Build:**
- `tsconfig.json` - TypeScript configuration with path aliases (`@/*` → `./src/*`)
- `next.config.ts` - Empty Next.js configuration (defaults used)
- `eslint.config.mjs` - ESLint configuration
- `tailwind.config.[js|ts]` - Tailwind configuration (via @tailwindcss/postcss)

## Platform Requirements

**Development:**
- Node.js (current LTS or later)
- npm (included with Node.js)
- Modern browser with ES2017+ support

**Production:**
- Deployment target: Vercel (preferred, as per Next.js documentation)
- Can also deploy to any Node.js hosting environment
- Next.js 16.2.2 requires Node.js 18.17+ on production servers
